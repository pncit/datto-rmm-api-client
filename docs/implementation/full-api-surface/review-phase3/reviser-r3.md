| architect-r2-f1 | Fixed | `scrub`/`scrubEntries` in `src/logging/mask.ts` now thread a `Set<object>` of live recursion ancestors. On entry to an array or plain object, the value is added to `seen`; if already present (a genuine ancestor cycle, e.g. `req.self = req`), recursion short-circuits and returns the `"[circular]"` sentinel instead of recursing; the value is removed from `seen` in a `finally` on exit, so the same object reached via two independent (non-circular) branches is still walked in each — only a true cycle is short-circuited, not a benign shared reference. `scrubMeta` starts a fresh `seen` set per top-level call. Added three regression tests in `tests/unit/logging/mask.test.ts`: a self-referential plain object under a non-UDF key no longer throws and resolves to `"[circular]"` at the cycle point (sibling `udf1` still masked); a circular array does the same; and a non-circular object shared across two sibling keys is still fully scrubbed in both places (proves the guard doesn't over-suppress a DAG). |
| architect-r2-f2 | Fixed | Rewrote the `withUdfMasking` docstring in `src/logging/mask.ts` to drop the absolute "no call site — current or future — can leak an unmasked UDF value" claim and instead state the guarantee's two actual bounds: only `meta` (never the message string) is scrubbed, and only plain-object/array structure inside `meta` is walked — a `Date`/`Map`/`Error`/other class instance is returned unmasked. Named the concrete risk: `DattoApiError#response` holds a raw wire body, so logging `{ err }` with a caught `DattoApiError` does not redact any UDF value nested in `err.response`, and call sites must not rely on `meta` scrubbing raw wire payloads embedded in non-plain objects — they should extract only the needed fields as plain data instead. |
| project-lead-r2-f1 | Fixed | `wrap` in `src/logging/mask.ts` now branches on `meta === undefined`: an omitted `meta` forwards to `logger[method](message)` (one argument), and a supplied `meta` forwards `logger[method](message, scrubMeta(meta))` (two arguments) — no more explicit `undefined` second argument on a no-meta call. Updated the existing "passes calls through unchanged when no meta is supplied" test to assert `toHaveBeenCalledWith("no meta here")` and that the recorded call has exactly one argument. Added a new test that wraps the real `console` (spying on `console.info`) and confirms a no-`meta` call reaches it as a single argument, closing the gap the finding noted (`mask.test.ts` previously only asserted against a `vi.fn()` sink, which doesn't observe `console`'s variadic-print behavior). |
