| typescript-cop-r2-f1 | Fixed (via a verified alternate mechanism — see note) | Added closed (no index-signature) `StrictSchemaNode`/`StrictOpenApiOperation` types in new `tests/generated/strict-fixture-types.ts`, and applied them via the `satisfies` operator to every hand-written schema-node/operation literal in `patch-spec.test.ts` and `widen-enums.test.ts` (`buildValidSpecFragment`'s schemas, every `withOperation(...)` call's operation argument, every new-schema assignment in the orphan-pruning tests, and every inline spec literal in `widen-enums.test.ts`, including the new content-type test cases above). A typo (e.g. `respnoses` for `responses`) in any of these now fails `npm run typecheck:test` with `TS2353`/`TS2561` — verified directly by temporarily introducing several such typos and confirming each one is caught, then reverting. **Deviation from the literal recommendation, documented:** the recommendation also said to drop `patch-spec.test.ts`'s local `SpecFragment`'s own `[key: string]: unknown` and use the closed type as `buildValidSpecFragment()`'s explicit return-type annotation. I verified with throwaway probes that this does not compile: TypeScript requires a *named* interface to declare its own index signature to be assignable to a target index-signature type (even when the target's index value is `unknown`) — only a fresh, unannotated object literal is exempt from that requirement. A `SpecFragment`-without-index-signature return type would therefore fail every `patchSpec(spec)` call in the suite with "Index signature for type 'string' is missing," and `withOperation`'s two-level computed-key (`[pathKey][method]`) merge would fail the same way if its `operation` parameter were narrowed to the closed type. `satisfies` achieves the identical excess-property-checking guarantee (confirmed by the typo probes above) without that regression, so I used it in place of the suggested explicit-annotation mechanism; `SpecFragment`/`SchemaNode`/`OpenApiOperation` container types are otherwise unchanged. Ran the full `npm run typecheck` / `npm test` / `npm run generate` + reproducibility gate afterward — all green. |
