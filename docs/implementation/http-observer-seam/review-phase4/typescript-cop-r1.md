| typescript-cop-r1-f1 | Medium | Open | TypeHole | `tests/integration/http-observer.test.ts:66-68` (`eventsOf`), used at `:93`, `:101`, `:146`, `:149`, `:168` | `eventsOf` filters `ObservedEvent[]` by `kind` with a plain `(e) => e.kind === kind` predicate (not a type-predicate function), so `Array.prototype.filter` cannot narrow the discriminated union and the mapped `.event` result stays typed as the full `DattoHttpRequestEvent \| DattoHttpResponseEvent \| DattoHttpErrorEvent` union. Every call site then papers over this with an `as DattoHttpRequestEvent[]` / `as DattoHttpResponseEvent[]` / `as DattoHttpErrorEvent[]` cast — five casts whose correctness depends entirely on `kind` being spelled right at each call site, which is exactly the kind of typo the compiler should catch and currently cannot. | Make `eventsOf` generic and narrow with a type-predicate filter so each call site's result is correctly typed with no cast: `function eventsOf<K extends ObservedEvent["kind"]>(events: ObservedEvent[], kind: K): Extract<ObservedEvent, { kind: K }>["event"][] { return events.filter((e): e is Extract<ObservedEvent, { kind: K }> => e.kind === kind).map((e) => e.event); }`. Then drop the `as …[]` cast at every call site. |
| typescript-cop-r1-f2 | Medium | Open | TypeHole | `tests/integration/http-observer.test.ts:199-205` | After asserting `terminal[0]!.kind`/`terminal[1]!.kind` via `expect(...).toBe(...)` (which does not narrow for the type checker), the test reaches into `.event` via `(terminal[0]!.event as DattoHttpErrorEvent)` and `(terminal[1]!.event as DattoHttpResponseEvent)` — casts standing in for a real narrow, so a future reordering of the `terminal` array (e.g. if a scenario changes to `[onResponse, onError]`) would silently mis-assert the wrong event shape instead of failing to compile. | Narrow with an `if` on `.kind` (which the compiler *can* follow for a discriminated union) instead of casting, e.g. `const [first, second] = terminal; if (first!.kind !== "error") throw new Error("expected error event first"); expect(first.event.statusCode).toBe(429);` — same for `second`. |
| typescript-cop-r1-f3 | Medium | Open | TypeHole | `tests/integration/http-observer.test.ts:96` | `DattoHttpRequestEvent.body` is deliberately typed `unknown` (it can be a serialized urlencoded string or a pre-serialization object, per `http-observer.ts`'s own doc comment). The test casts it straight to `string` (`grantRequest!.body as string`) with no runtime check, so if the grant body's shape ever changed (or a future refactor sent a plain object instead of the serialized form), `new URLSearchParams(...)` would silently accept the wrong shape (it also accepts records/iterables) rather than the test failing with a clear message pointing at the real cause. | Assert the runtime shape before casting: `if (typeof grantRequest!.body !== "string") throw new Error("expected serialized urlencoded grant body");` then use `grantRequest.body` (now narrowed to `string`) with no `as`. |
