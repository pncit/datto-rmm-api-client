## reviser — round 1

| ID | Disposition | Rationale (and, for Fixed, what changed) |
|----|-------------|-------------------------------------------|
| implementation-auditor-r1-f1 | Fixed | Replaced `observerCallbackSchema`'s `z.function({ input: [z.any()], output: z.void() })` with a non-wrapping, shape-only `z.custom<(event: never) => unknown>((v) => typeof v === "function")` in `src/http/http-observer.ts`. `z.custom` returns the validated input unchanged, so `dattoHttpObserverSchema.parse({ onRequest }).onRequest === onRequest` is now `true` (verified), and a parsed callback that returns a value or an about-to-reject thenable no longer throws a synchronous `ZodError` at call time — `invokeObserver` (`src/http/observer.ts`, unchanged) now owns return-value/async-rejection tolerance exactly as R7 requires. `.strictObject`, `.optional()`, and the existing non-function/unknown-key rejection behaviors are preserved. Rewrote the doc comment above `observerCallbackSchema` to explain why the `z.function`/`dattoLoggerSchema` form is incompatible here (rather than claiming to mirror it), and added a deviation entry to `implementation-phase1-notes.md` §5 recording the empirical basis for the change. `dattoLoggerSchema` itself is untouched (out of Phase 1 scope; its usage does not trigger the defect). Verified via `npx tsc --noEmit` (clean) and `npm test` (558/558 passing, including the new f2 regression tests below); re-ran the Phase 1 exit-gate greps unchanged (all pass). |
| implementation-auditor-r1-f2 | Fixed | Added three tests to `tests/unit/http/observer.test.ts` under a new `describe("invokeObserver on a schema-parsed callback (R7 regression — f1/f2)")` block, each obtaining the callback **through** `dattoHttpObserverSchema.parse(...)` rather than a hand-built raw `fn`: (1) a value-returning `onRequest` (`(event) => received.push(event)`, an `Array#push` idiom that returns a number) invoked via `invokeObserver` produces zero `warn` calls and still delivers the event; (2) an `async`/rejecting `onResponse` invoked via `invokeObserver`, monitored with a real `process.on("unhandledRejection", ...)` listener, produces zero unhandled rejections and exactly one `warn` naming `onResponse`; (3) `dattoHttpObserverSchema.parse({ onRequest }).onRequest` is identity-equal (`toBe`) to the supplied `onRequest`. All three tests were confirmed to fail against the pre-fix `z.function`-based schema (reproduced manually — the value-returning case throws synchronously, and the async case throws before the thenable is returned, both surfacing as the wrong `warn` count / an unhandled rejection) and pass after the f1 fix. |
| implementation-auditor-r1-f3 | Fixed | In `tests/unit/client/config.test.ts`'s "still invocable after parsing" case, changed `expect(received).toEqual([rawEvent])` to `expect(received).toHaveLength(1); expect(received[0]).toBe(rawEvent);` — asserting identity rather than structural equality, so the test now pins that parsing neither clones nor substitutes the delivered payload/callback, matching what the notes already claimed. |
