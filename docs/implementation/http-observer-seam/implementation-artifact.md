# HTTP Observer Seam ŌĆö Implementation Process Artifact

_This cycle implemented an optional, pure-observer `httpObserver` seam on the Datto RMM client that hands a consumer the raw request/response/error of every outbound HTTP exchange ŌĆö without exposing axios ŌĆö restoring a compliance-observability capability that the 1.0.x transport rewrite had removed._

## Genesis

The 1.0.x rewrite took ownership of the HTTP transport: the client builds its own axios instances, wires authentication, rate limiting, retry, and pagination into them, and the strict config schema hard-rejects a caller-supplied `axiosInstance`. That was a deliberate isolation decision, but it removed a capability some 0.1.x consumers depended on. In 0.1.x a consumer could inject an axios instance and register interceptors to emit a redacted observability artifact for every Datto RMM HTTP exchange (URL, method, headers, body, status, duration, error) into a compliance/audit pipeline ŌĆö covering the internal OAuth token-grant call and every pagination page. In 1.0.2 the only observability surface was `DattoLogger`, which is deliberately body/header-free and UDF-masked, so it could not carry the raw payloads an audit artifact needs. A compliance-bound consumer therefore could not adopt 1.0.x without forking the transport or staying on 0.1.x.

The design answer was a transport-agnostic **HTTP observer seam**: optional `onRequest`/`onResponse`/`onError` callbacks grouped under `httpObserver` that fire once per physical HTTP attempt, deliver raw (un-redacted) wire-fidelity payloads, and cannot alter, delay, or fail a request ŌĆö while axios stays entirely internal. This is the HTTP-layer analogue of the structured `DattoLogger` and `DattoApiError` surfaces 1.0.x already established.

## Outcome

The seam shipped as designed. `DattoRmmClientConfig` now accepts an optional `httpObserver`; the client exports exactly **five** axios-free public types ŌĆö `DattoHttpObserver`, `DattoHttpRequestEvent`, `DattoHttpResponseEvent`, `DattoHttpErrorEvent`, and the shared `DattoHttpHeaders` alias. Both transport layers are instrumented through one internal helper module (`src/http/observer.ts`) so they cannot drift: the shared axios instance (`createHttpClient`) via an observer request interceptor registered first (so it runs last under axios LIFO, post-throttle/post-auth) plus firing from the fulfilled response handler and `handleResponseError`; and the OAuth grant path (`AuthManager.performRefresh`) at its own dispatch point. Request fields are captured-and-stashed on the per-attempt request config (the `axios-augment.d.ts` `__dattoObserverCapture` precedent) so terminal events reuse the exact form `onRequest` observed. The observer is threaded raw/unmasked (never through `withUdfMasking`); a callback `throw` or returned rejection is swallowed and logged once at `warn`. The living design and plan are `docs/implementation/http-observer-seam/design.md` and `plan.md`.

The work landed in four phases (commits `faf57e9`, `f00f253`, `a3e3286`, `1b02b54`), each passing its exit gate; the final suite reached 590 passing tests, and `dist/index.d.ts` contains the five types with no `declare module "axios"` and no axios type in the observer signatures.

## Process at a glance

Four implementation phases, each reviewed by up to five reviewers (architect, engineer, project-lead, typescript-cop, implementation-auditor) with a triage/mediator and reviser, most converging in two rounds:

- **Phase 1** (public types, schema, internal helper) ŌĆö the heaviest review. A pre-review defect (the wrapping `z.function` callback schema) was already fixed before round 1; that fix then caused a collateral regression three reviewers converged on in round 2 (the `never`-erasure of callback types). One design decision from Phase 1 carried a "chain watch" for a third round that never materialized.
- **Phase 2** (shared axios instance) ŌĆö one High finding on URL fidelity escalated to a human ruling (the run's second design amendment); the rest was test hardening.
- **Phase 3** (OAuth grant path) ŌĆö all findings Low; one triggered a design amendment (grant URL composition) run through the amendment/design-auditor loop.
- **Phase 4** (assembled-client integration tests + README) ŌĆö no production code; one High requirements-coverage gap and a cluster of test type-safety and documentation-accuracy fixes.

Two findings were escalated to the human for a ruling; three design amendments were requested across the run (two applied, one declined as redundant). All findings closed; no reviewer left an open finding at convergence.

## Key findings

**The callback schema churned twice, then stabilized (Phase 1).** The plan mandated mirroring `dattoLoggerSchema`'s `z.function({ output: z.void() })` shape-only validation. The implementation-auditor proved empirically (against the installed `zod@4.4.3`) that this form does **not** pass the callback through: it returns a validating proxy that is not identity-equal to the supplied function and validates the return value against `void` at call time. Consequences directly defeated R7/R9: a value-returning callback (the idiomatic `(e) => buffer.push(e)`) threw a synchronous `ZodError` mis-reported as a callback failure, and an async callback threw before its thenable was returned, leaking an unhandled rejection. The fix replaced it with a non-wrapping `z.custom` validator (identity pass-through; `invokeObserver` owns return/async tolerance). That fix, however, collapsed all three callbacks into one field-agnostic `z.custom<(event: never) => unknown>`, which erased the concrete event types out of the directly-exported `DattoRmmClientConfig` ŌĆö the idiomatic inline `httpObserver: { onRequest: (event) => event.method }` failed to type-check with `event: never`. Architect, project-lead, and typescript-cop independently flagged this (three High findings, same root cause). The resolution was a per-field generic factory `observerCallbackSchema<Fn>()` keyed off `DattoHttpObserver["on*"]`, restoring per-field inference while keeping the identity pass-through, plus compile-time `surface-pin.ts` regression pins asserting no-`never` inference and schemaŌåöinterface parity. The triage explicitly warned that a third round on this same schema shape would escalate rather than spot-fix; it converged instead.

**Observed URL fidelity ŌĆö the recurring theme, resolved twice (Phases 2 and 3).** The shared-instance observer initially composed the observed `url` by manual `` `${baseURL}${url}` `` concatenation, which drops `requestConfig.params` (axios serializes them into the query string only after request interceptors run) and diverges from axios's own `combineURLs`/`buildFullPath` joining. Every filtered list GET and pagination's first page (whose cursor rides in `params:`) was observed with its entire query string missing ŌĆö violating the "exactly as dispatched" contract at the heart of the audit use case. This was a High finding that also *overrode a previously-ruled, design-pinned decision*, so it escalated to the human. The same theme recurred in Phase 3 on the grant path, where manual `` `${apiUrl}${GRANT_PATH}` `` concatenation double-slashes under a trailing-slash `apiUrl` (`z.url()` permits one) while axios dispatches a single slash. Both sites were converged on `instance.getUri(...)` / `grantClient.getUri({ url: GRANT_PATH })`, making the two instrumentation sites resolve URLs identically through axios's own resolver.

**Wire-fidelity and DRY hardening of the shared helper (Phases 1 & 3).** `captureRequest`'s `?? "get"` let an empty-string method through as `""`, contradicting the "uppercased, non-empty" contract ŌĆö changed to `||`. `normalizeHeaders` re-spelled the `DattoHttpHeaders` alias inline ŌĆö cast through the named alias instead. `invokeObserver`'s swallow paths called `logger?.warn` unguarded, so a throwing consumer `warn` could re-open the exact R7 hole the module exists to close ŌĆö wrapped in a `safeWarn` helper. The grant's `Content-Type` literal was written twice (constructor header and capture) with nothing tying them ŌĆö hoisted to one `GRANT_CONTENT_TYPE` constant.

**Test-coverage and type-safety gaps.** Two of the seven Phase-1 helper primitives (`fireRequest`, `fireResponse`) had no direct unit test despite the phase charter claiming isolation coverage ŌĆö added. Phase 2 lacked a 401/`onUnauthorized` transparent-retry observer test (the one branch firing through a different retry mechanism) ŌĆö added, pinning `onRequest, onError(401), onRequest, onResponse(200)`. Phase 3 lacked single-flight (N concurrent `getToken()` ŌćÆ one event pair) and error-path terminal-exclusivity coverage ŌĆö added. Test event captures repeatedly leaned on `as DattoHttp*Event` casts instead of discriminated-union narrowing (Phases 2 and 4) ŌĆö replaced with tuple types / type-predicate filters / `kind`-narrowing guards.

**Documentation accuracy (Phase 4).** The README/JSDoc raw-credential warning enumerated the bearer token and API key but omitted the **API secret** (the grant's `password` field ŌĆö the most sensitive of the three) ŌĆö added to both. The terminal-event bullets said they were "adding" fields, hiding that the request-side fields are *renamed* to `requestHeaders`/`requestBody` (a consumer would write `e.headers` and get `undefined`) ŌĆö rewritten. The "Upgrading from 0.1.x" note removed `axiosInstance` with no pointer to its replacement ŌĆö a cross-reference to the observer section was added.

## Key decisions

- **URL composition escalated to the human (Phase 2).** The High URL-fidelity finding both exceeded the mediator's Medium/Low authority and overrode a settled, design-pinned decision (the earlier `architect-r1-f2` ruling had settled relative-vs-absolute host inclusion but never considered query-string fidelity). The human chose **Option A**: adopt `instance.getUri(requestConfig)`, amend `design.md` and `plan.md` to specify the `getUri`/serialized-params composition, and add `params` + paginated-first-page tests ŌĆö over the alternative of accepting the limitation and narrowing the contract wording to drop "exactly as dispatched." This preserved the audit contract. The subsumed slash-joining/absolute-URL finding was closed by the same fix.

- **The grant URL amendment was applied; its duplicate was declined (Phase 3).** The architect's grant-URL finding was ruled a **Design Change** and run through the amendment loop (commit `67a6060`): `design.md` Decision 5, the payload comment, and the Success Criteria bullet were amended to mandate `grantClient.getUri({ url: GRANT_PATH })`; the design-auditor ratified it against `axios@1.18.1`'s actual `getUri`/`combineURLs` behavior. The engineer's twin finding requested a *second* design edit for the same normalization; because the design was already corrected, that amendment was **declined as redundant** ŌĆö the declination rejected the duplicate design edit, not the underlying one-line code fix, which the single architect-driven change already delivered.

- **`onError.error` is `unknown`, handed off raw ŌĆö never re-mapped.** Reaffirmed throughout: the seam hands off the request error exactly as the transport produced it (an `AxiosError`, a `DattoApiError`, or anything else) because a `throw` guarantees nothing about shape, and re-deriving a `DattoApiError` in the observer risks diverging from what the client actually throws (e.g. a terminal 429's `buildRateLimitError` the observer cannot reproduce without retry context). No `mapObserverError` exists; `fireError` adds `statusCode`/response fields only when a response is present. Reviewers verified at the transport level that observed errors are `axios.isAxiosError(...) === true` and `not.toBeInstanceOf(DattoApiError)` while the SDK still throws its mapped error to the caller.

- **Raw/unmasked delivery is a conscious exemption from the masking boundary.** The observer is threaded raw into both transport layers, never through `withUdfMasking` ŌĆö an explicit divergence from the `DattoLogger` masking guarantee, justified because the consumer's redactor needs wire fidelity (including bearer tokens and the API key/secret). The masked logger is used *only* to report a swallowed callback failure. Both wiring sites carry doc comments contrasting the raw `httpObserver` with the adjacent masked `logger`.

- **Terminal-event selection keys on the physical HTTP status, and capture is independent of which callbacks are supplied.** `onResponse` fires for a 2xx, `onError` for any non-2xx or no response; each dispatched attempt fires exactly one terminal event plus one `onRequest`. Placing the shared-instance `onError` *after* `handleResponseError`'s `!axios.isAxiosError` guard is what excludes the two non-dispatched paths (rate-limiter reject and lazy Bearer `getToken()` throw) from double-reporting a grant failure. Capture-and-stash runs whenever `httpObserver` is present ŌĆö not gated on `onRequest` ŌĆö so an `onError`-only consumer still gets populated request fields on its terminal event; the plan and tests pin this. The grant fires `onResponse` on the 2xx *before* `tokenResponseSchema.safeParse`, so a malformed-token 2xx fires exactly one terminal event (`onResponse`) and never `onError`, even though `performRefresh` throws.

- **Minor scope/deviation judgments.** The Phase-1 `dattoLoggerSchema` (which carries the same latent `z.function` form) was deliberately left untouched ŌĆö out of scope and not triggered by its internal, void-returning usage. A reviser deviation in Phase 4 used three concrete `kind`-keyed overloads for the `eventsOf` test helper instead of the triage's suggested single generic signature, because the generic form does not compile under the repo's `tsc` (a known indexed-access-over-conditional-type limitation on a 3-member union) ŌĆö accepted by reviewers as achieving the identical outcome (zero casts; a `kind` typo fails to compile).

## Known limitations

- **Deferred by design (not defects):** correlation/attempt identifiers on the payload and a bundled redacting observer implementation are consciously out of scope ŌĆö the current consumer's artifact is per-attempt and self-contained, and it redacts on its own side and wants raw delivery. Both are recorded in the design's Future Considerations to revisit on concrete demand.
- **Accepted header limitation:** the grant call's `Authorization: Basic` header (the non-secret `public-client:public` pair, applied by axios internally from the per-request `auth:` option) is **absent by design** from the captured header map. This is intentional and documented; the security-relevant credential (the API key/secret) rides in the captured body and is verified present. Phase 4 added an end-to-end assertion that the grant request headers omit `authorization`/`Authorization`.
- The seam observes only HTTP exchanges. Post-2xx and non-HTTP failures ŌĆö response schema validation (`DattoValidationError`), the grant's malformed-token `DattoApiError`, and pagination cursor/guard failures ŌĆö deliberately do **not** fire `onError`; the attempt that received the 2xx already fired `onResponse` with the raw body.
